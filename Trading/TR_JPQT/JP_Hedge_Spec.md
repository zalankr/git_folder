## 2. 유니버스 및 자산 정의

### 2.1 주식 자산: JPQT 방식 일본 개별주 바스켓 (10종목 동등가중)

**스크리닝 유니버스**: TOPIX 500 또는 TOPIX Core 30 + Large 70 + Mid 400
- 1차 필터: 시가총액 상위 500개
- 2차 필터: 일평균 거래대금 5억엔 이상
- 3차 제외: 금융섹터(은행/증권/보험) 일부 제외 옵션

**JPQT 팩터 정의 (월간 리밸런싱 시 계산)**
```
Quality   = 25% ROE(TTM) + 25% ROIC(TTM) + 25% 영업이익률(TTM) + 25% (1 - 부채비율)
Value     = 33% 1/PER + 33% 1/PBR + 33% (EBITDA/EV)
Momentum  = 50% 12M-1M 수익률 + 50% 6M 수익률
Low-Vol   = 1 / (252일 일간수익률 표준편차)
```

**종합 스코어**: 각 팩터를 유니버스 내 Z-score 표준화 후
`Score = 0.30·Q + 0.25·V + 0.30·M + 0.15·LV`

**선정 규칙**:
- 종합 스코어 상위 10종목
- 한 섹터(GICS Sector) 최대 3종목 제한 (집중도 완화)
- 동등가중 (1종목당 10%)

**데이터 소스 옵션**:
1. yfinance: 기본 OHLCV + 일부 펀더멘털 (무료, 정확도 보통)
2. KIS API 해외종목 정보 조회 + 별도 펀더멘털 소스 결합
3. Stooq, J-Quants API(일본 거래소 공식) — 펀더멘털은 별도 확보 필요

권장: 초기엔 yfinance + pyKRX 일본판/J-Quants 무료 티어 조합, 운영 안정화 후 유료 데이터 검토.

### 2.2 헤지 자산: 일본 상장 ETF

| 자산군 | 1차 후보 | 종목코드 | 비고 |
|---|---|---|---|
| 금 | Nomura NF Gold Price ETF | **1328.T** | 엔화표시 금가격 추종, AUM 충분, 2010~ |
| 금 (대체) | Japan Physical Gold ETF | 1540.T | 실물 금 백킹, 2010-07~ |
| 채권 | iShares Core JGB ETF | **1482.T** | 7-10년 국채, 2016-05~ |
| 채권 (대체) | Nomura JP Bond NOMURA-BPI ETF | 2510.T | 종합 채권지수, 2017-12~ |

**기본 선택**: 금=1328.T, 채권=1482.T

## 3. 신호 정의 (월말 종가 기준)

### 3.1 기준 시계열: TOPIX ETF 1306.T (시장 전체 추세 판단용)
> 개별주 가격이 아니라 **시장 대표지수 1306.T**로 신호 판단. 개별주는 비중 배분 대상일 뿐.

### 3.2 신호 1: 200일 이동평균
```
MA200(t) = mean(Close[t-199:t+1])
Signal_MA = (Close_monthend > MA200_monthend) ? True : False
```

### 3.3 신호 2: 12개월 모멘텀
```
Mom12(t) = Close[t] / Close[t-252] - 1
Signal_MOM = (Mom12 > 0) ? True : False
```

### 3.4 점검 시점
- **월말 마지막 거래일 KST 15:30 (도쿄 거래소 마감 후)**
- 시간대 주의: 일본은 한국과 시차 0 (도쿄 = 서울)
- 거래소 휴장일은 직전 거래일을 월말로 처리

## 4. 비중 결정 매트릭스

### 4.1 3단계 상태 분류

| 상태 | 조건 | 주식 영역 | 금 영역 | 채권 영역 | 빈도(과거) |
|---|---|---:|---:|---:|---:|
| **Bull** | MA✓ AND MOM✓ | 80% | 20% | 0% | 57.4% |
| **Neutral** | MA xor MOM | 50% | 30% | 20% | 17.4% |
| **Bear** | MA✗ AND MOM✗ | 0% | 60% | 40% | 25.3% |

### 4.2 주식 영역 내부 배분 (JPQT 바스켓)
- 주식 영역 비중 W_stock 결정 후, 10종목에 동등가중
- 각 종목 비중 = W_stock / 10
- 예: Bull 상태에서 종목당 비중 = 80% / 10 = 8%

### 4.3 채권 ETF 부재 옵션 (운영 단순화)
USE_BOND = False 설정 시 채권 비중을 금으로 흡수:
| 상태 | 주식 | 금 | 채권 |
|---|---:|---:|---:|
| Bull | 80% | 20% | 0% |
| Neutral | 50% | 50% | 0% |
| Bear | 0% | 100% | 0% |

백테스트상 성과 차이 미미.

## 5. 매매 조건 및 실행 규칙

### 5.1 리밸런싱 트리거
**정기 리밸런싱**: 매월 마지막 거래일 종가 확인 후, 익영업일 09:00~10:00 KST 매매 집행

**조건부 리밸런싱**: 다음 중 하나 해당 시 즉시 재계산
1. 시장 상태(Bull/Neutral/Bear) 변경 감지
2. JPQT 종목 리스트 신규 진입/탈락 발생
3. 개별 종목 비중이 목표 대비 ±25% (상대 편차) 이상 이탈

### 5.2 발주 규칙
- **주문 유형**: 지정가만 사용 (시장가 금지)
- **매수 슬리피지**: 직전 종가 +0.5% 지정가
- **매도 슬리피지**: 직전 종가 -0.5% 지정가
- **분할 발주**: 1종목 1일 거래량의 5% 초과 시 3분할 (10:00 / 11:30 / 14:00 KST)
- **체결 미달 처리**: 종가 30분 전(15:00 KST) 미체결 잔량은 슬리피지 ±1.0%로 재발주
- **종가 마감 후 미체결**: 익영업일 동일 가격대로 재시도, 3영업일 누적 미체결 시 알림 후 수동 처리

### 5.3 매매 순서 (현금 부족 방지)
1. **매도 우선**: 비중 축소/제거 대상 먼저 전량 매도
2. **체결 대기**: 매도 체결 확인 후
3. **매수 집행**: 신규/증액 대상 매수
4. 매도-매수 간 최소 30분 간격 (체결 안정성)

### 5.4 KIS API 거래 코드 (해외주식 일본)
- 거래소 코드: `TKS` (도쿄)
- 통화: JPY
- TR_ID 매핑 (운영서버 기준):
  - 매수 주문: `TTTS1002U`
  - 매도 주문: `TTTS1001U`
  - 잔고 조회: `CTOS5002R`
  - 체결 조회: `CTOS5004R`
- ※ 실제 TR_ID는 KIS OpenAPI 문서 최신본 재확인 필수 (변경 가능성)

### 5.5 호가단위 (TSE)
| 가격대 (JPY) | 호가단위 |
|---|---:|
| ~3,000 | 1 |
| 3,000~5,000 | 5 |
| 5,000~30,000 | 10 |
| 30,000~50,000 | 50 |
| 50,000~ | 100 |

지정가 발주 시 호가단위 반올림 함수 필수.

### 5.6 매매 수량 단위
- 대부분 일본 주식: 100주 단위 (단원주)
- ETF: 1주 단위 (1306, 1328, 1482 등 단원 = 1주)
- 발주 수량은 100주 단위로 내림 처리 (개별주의 경우)

## 6. 분할/배당/수수료 처리

### 6.1 분할(액면분할) 대응
- yfinance 분할 자동반영 미스 사례 존재 (예: 1306.T 2026-03-30 1:10 분할 미반영)
- **방어 코드**: 일간 수익률 절댓값 30% 초과 시 분할 의심 → Telegram 경보 + 수동 검증
- 신호 계산 시 가격 시계열 일관성 검증 함수 필수

### 6.2 배당
- 일본 ETF는 연 1~2회 배당 (1306은 7월, 1월)
- 배당락 당일 가격 갭은 신호에 영향 없도록 수정주가 사용
- 배당금 입금 후 다음 리밸런싱 시 비중에 반영 (별도 재투자 매매 불필요)

### 6.3 거래비용 가정
- 매매 수수료: KIS 일본 주식 약 0.25% (편도) — 실제 적용 수수료 확인 필요
- 환전 비용: JPY 계좌 운영 권장 (매매 시마다 환전 회피)
- 백테스트 가정: 편도 0.10%로 충분히 보수적 (실제는 더 작을 수 있음)

## 7. 운영 스케줄 (crontab)

```cron
# 시각은 KST 기준
# 매일 16:00: 일간 신호 모니터링 (Bull/Bear 상태 변화 감지용)
0 16 * * 1-5  /usr/bin/python3 /var/autobot/JP_Hedge.py --mode=monitor

# 매월 말일 16:00: 월말 신호 확정 및 비중 산출
0 16 28-31 * *  /usr/bin/python3 /var/autobot/JP_Hedge.py --mode=signal --check-month-end

# 매월 1영업일 09:05: 리밸런싱 발주 시작
5 9 1-3 * 1-5  /usr/bin/python3 /var/autobot/JP_Hedge.py --mode=execute

# 매일 15:30: 미체결 잔량 점검 및 재발주
30 15 * * 1-5  /usr/bin/python3 /var/autobot/JP_Hedge.py --mode=cleanup
```

월말 판정 로직: 해당 일자 +1영업일이 다음 달인 경우 "월말 마지막 거래일"로 인식.

## 10. 백테스트 검증 사양

### 10.1 백테스트 모듈 별도 분리
```
backtest/
├── bt_engine.py        # NAV 계산 엔진
├── bt_jpqt.py          # JPQT 바스켓 백테스트
├── bt_full_strategy.py # 헤지전략 통합 백테스트
└── results/
```

### 10.2 검증 시나리오
1. **Baseline**: 1306.T 100% 바이앤홀드
2. **Strategy-A**: 1306.T + 1328.T + 1482.T (개별주 없이 ETF만, 본 백테스트와 동일)
3. **Strategy-B**: JPQT 바스켓 + 1328.T + 1482.T (목표 전략)
4. **Strategy-C**: JPQT 바스켓 100% (헤지 없음, JPQT 알파만 확인)

### 10.3 평가 지표
CAGR, 연 변동성, Sharpe, MDD, Calmar, 연도별 수익률, 월간 승률, 턴오버, 추정 거래비용 차감 후 순수익률

### 10.4 견고성 테스트
- 채권 ETF 데이터 짧음(1482는 2016~) → 채권 부재 구간은 금으로 흡수해서 비교
- JPQT 룩백 윈도우 변경 (12M-1M vs 6M, 3M)
- 리밸런싱 빈도 변경 (월 1회 vs 분기 1회)
- Look-ahead bias 방지: 신호 t시점 데이터로 t+1일 발주 가격 사용

## 11. 리스크 및 운영 주의사항

### 11.1 데이터 리스크
- yfinance 일본 종목 데이터 안정성 보통 (분할 누락 사례 있음)
- 월말 신호 산출 후, 익영업일 발주 전에 **데이터 정합성 재확인 단계** 필수
- 펀더멘털 데이터 적시성: 분기 실적 발표 지연 가능 → 90일 lag 적용

### 11.2 시장 구조 리스크
- 도쿄증권거래소 휴장일은 별도 (한국 휴장일과 다름) — `pandas_market_calendars` 의 'JPX' 사용
- 단원주 100주 제약으로 인해 종목당 비중 오차 발생 가능 → 1% 이내면 무시
- 거래량 얕은 종목은 슬리피지 확대 가능 → 일평균 거래대금 5억엔 미만 종목은 유니버스에서 제외

### 11.3 환율 리스크
- 본 전략은 **JPY 베이스**로 운용 (원화 환산 손익 별도 계산)
- 한국 거주자 관점: 엔/원 환율 변동도 별도 위험 요인 → 환헤지 여부 선택사항

### 11.4 세금
- 일본 주식 양도소득: 한국 거주자 기준 해외주식 양도소득세 22% (250만원 공제)
- 배당: 일본 원천징수 + 한국 종합소득 합산
